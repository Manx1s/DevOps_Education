# DevOps_Education

№1

Задача:
Высоконагруженная БД MySQL, критичная к отказу.
Используемый тип платформы: Физические сервера.

Причина выбора: БД MySQL требует максимальной производительности (выполнение запросов клиент-сервер). Требуется минимальная задержка к действия ввода-вывода и скорости работы дисков. 

Задача:
Различные web-приложения.
Используемый тип платформы: Паравиртуализация

Причина: Возможность быстро переносить приложение между другими виртуалками (масштабируемость), легко изолировать и использовать ресурсы сервера.

Задача:
Windows-системы для использования бухгалтерским отделом
Используемый тип платформы: Виртуализация уровня ОС

Причина: Возможность быстро создать рабочее место и склонировать машину для другого пользователя.

Задача:
Системы, выполняющие высокопроизводительные расчёты на GPU.
Используемый тип платформы: Физические сервера

Причина: Расчеты на GPU памяти требуют обычно прямого доступа к видеокартам. Хоть и существуют технологии проброса GPU они снижают производительность относительно физического сервера с GPU.

№2
Выберите подходящую систему управления виртуализацией для предложенного сценария. Опишите ваш выбор.

Сценарии:

1) 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.
Возможно использование VirtualBox, но это крайне неудобный способ разворачивания сотни виртаульных машин.
Более удобный и производительный способ, хоть и платный - Hyper-V.

2) Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.
Если требуется разворачивание виртуальных машин на базе Linux - как опенсорсное решение подойдет KVM.
Для Windows - возможно использование VirtualBox или бесплатных версий VMware/Hyper-V.

3) Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.
VirtualBox или Vmware для домашнего использования. Но VirtualBox более легок в использвоании для локальных задач

4) Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.
VMware/VirtualBox - Легок в разворачивании на локальном компьютере.

№3

Опишите возможные проблемы и недостатки гетерогенной среды виртуализации (использования нескольких систем управления виртуализацией одновременно) и что необходимо сделать для минимизации этих рисков и проблем. Если бы у вас был выбор, создавали бы вы гетерогенную среду или нет?

Сложность в управлении, возможные конфликты при работе такой среды виртуализации. Недостаток документации при совместной работе такой среды.
Сложность или несовместимость миграции ресурсов -> Снижение производительности.

Для снижения рисков необходимо определиться каким стеком необходимо пользоваться. Если такая среда уже есть, стоит мигрировать в одну среду виртуализации.
